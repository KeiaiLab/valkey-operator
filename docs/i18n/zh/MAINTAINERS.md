# 维护者 (Maintainers)

> English version: [MAINTAINERS.md](../../MAINTAINERS.md)

本文档跟踪对 `keiailab/valkey-operator` 拥有决策权的维护者。

## 当前维护者

| 名称 / 团队 | GitHub | 角色 | 领域 |
|---|---|---|---|
| keiailab maintainers | [@keiailab/maintainers](https://github.com/orgs/keiailab/teams/maintainers) | Lead | 全部 |

GitHub team `@keiailab/maintainers` 对项目所有领域拥有 merge 与 approval 权限。新增个别维护者按下述流程进行。

## 资格

贡献者在持续满足以下条件至少 6 个月后,可被提名为维护者:

- 20+ 条有意义的代码或文档 merged PR
- 30+ 条带建设性反馈的 PR review
- 切实遵守 [GOVERNANCE.md](../../../.github/GOVERNANCE.md) 与 [CODE_OF_CONDUCT.md](../../../.github/CODE_OF_CONDUCT.md)
- 对至少一个核心领域有深入理解 — controller、resource builders、restore / backup、cluster sharding、observability 等

## 新增维护者

1. 已有维护者 (或候选人本人) 开启提议新增的 issue 或 ADR。
2. `@keiailab/maintainers` 在 7 天评论窗口内进行 lazy consensus。
3. 如果没有反对意见,贡献者会被加入 GitHub team,并由后续 PR 更新 `MAINTAINERS.md`。

## 非活跃维护者

连续 6 个月没有活动的维护者将被转入 emeritus 状态 (权限收回,荣誉性列表保留)。回归路径与新增维护者流程一致。

## 领域覆盖 (与 CODEOWNERS 保持同步)

按目录的 reviewer 路由配置请参阅 `.github/CODEOWNERS`。
