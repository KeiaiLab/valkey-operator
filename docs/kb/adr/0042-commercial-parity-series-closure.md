# ADR-0042: Commercial Parity Series 종합 — 23 PR 시리즈 closure

- Date: 2026-05-10
- Status: Accepted
- Authors: @eightynine01

## Context

ADR-0040 (Helm chart vs Operator 채택 정책) 의 5 gap 해소 epic (#37) 으로 시작
한 commercial parity 작업이 *23 PRs (#38-#60)* 으로 확장. 최초 5 gap 외에 ADR-0027
(HPA deferred), ADR-0040 §6 미달 항목, 그 외 production hardening 다수 GA.

epic #37 은 5 sub-issue close 로 종료됐으나 *후속 13 PR* 은 별도 추적 없이
진행. 본 ADR 은 시리즈 전체 의사결정 + 각 PR 의 ADR-0040 §parity 평가축
대비 기여를 보존.

## Decision

23 PR 시리즈를 *single commercial parity epic* 으로 사후적으로 묶고, ADR-0040
의 후속 ADR 로 본 0042 를 채택. 각 PR 의 commercial parity 기여를 다음 매트릭스
로 추적:

## Consequences

**긍정:**
- valkey-operator 가 *enterprise tier 외 모든 commercial requirement* 충족.
  VolumeSnapshot full cycle / PITR phase 1 / SLO alerts / HA defaults / encryption
  enforce 모두 GA.
- 23 PR 모두 atomic (1 commit + 1 ship + 1 deploy) 정책 일관 — 회귀 위험
  최소.
- ADR-0027 (HPA deferred) → 실 구현. 다른 deferred ADR 들도 동일 패턴 가능.

**부정:**
- 23 PR 시리즈가 *epic #37 closure 후* 진행되어 추적 distributed.
  본 ADR 이 retroactive 통합 SSOT.
- 시리즈 종료 시점 (PR #60) 후 추가 PR 가 들어오면 본 ADR 갱신 의무.
  대안: ADR-0042 status=Implemented + 후속 ADR-NNNN 작성.

## Alternatives Considered

1. **별도 epic 신규 생성 + 13 PR re-link**: epic #37 가 close 됐으나 본 시리즈
   는 그 *연장선*. 새 epic 작성보다 ADR 통합이 의도-추적 명확. 거절.

2. **PR 별 별도 ADR**: 23 ADR 작성 — over-engineering. 각 PR 은 commit
   메시지 + parity-status 갱신으로 충분. 거절.

3. **Series 종료 명시 안 함**: 시리즈가 언제 끝났는지 모호. 향후 사용자/
   기여자 가 *어디서부터 enterprise-tier 였는가* 판단 어려움. 거절.

## 후속 (별도 epic 후보)

ADR-0040 §6 의 enterprise-tier 항목:

1. **PITR phase 2** — AOF replay-to-timestamp reconciler dispatch
2. **HPA for ValkeyCluster shard scaling** — slot 재분배 자동화
3. **Multi-pod VolumeSnapshot restore** — Replication / Cluster mode
4. **Active-Active multi-region** — 별도 product line

각 항목은 multi-day~multi-week epic 규모로 atomic PR 사이클 부적합.
