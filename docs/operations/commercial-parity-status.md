# Commercial Parity Status — valkey-operator

> 한국어 버전: [commercial-parity-status.ko.md](commercial-parity-status.ko.md)

Status after closing the five ADR-0040 ("Helm chart vs Operator
adoption") commercial-parity gaps, nine phase-2 hardening items,
and the CloudPirates valkey `0.20.2` cross-verification. This is
the inventory that establishes valkey-operator as a
**production-grade OSS Redis/Valkey operator**.

Comparison targets: the Bitnami Redis chart (post-paywall OSS) /
the CloudPirates Redis chart / Redis Enterprise / AWS ElastiCache /
GCP Memorystore.

The detailed values mapping for the CloudPirates valkey chart
`0.20.2` lives in
[`docs/operations/cloudpirates-valkey-compatibility.md`](cloudpirates-valkey-compatibility.md),
which is the **single source of truth** for that comparison.

This document answers, within one minute,
**"can I run production on this operator?"** Items that fall
short are marked **Pro tier / follow-up epic candidate**.

## 7. Decision matrix (ADR-0040 §4.1 update)

| Scenario | Recommendation |
|---|---|
| **Production now, small team, minimal external dependencies** | **CloudPirates chart** (simple, scratch-based security, avoids paywall) |
| **FIPS / CIS hardening as a regulatory requirement** | **Bitnami Secure Images (paid)** ($50K–$72K / yr) |
| **Cluster mode + auto-resharding + dynamic ops automation** | **valkey-operator** (current GA) — **unconditional recommendation** |
| **Replication + HA + auto-scaling** | **valkey-operator** — HPA + AutoFailover + automatic PDB / TSC defaults |
| **PITR + multi-region active-active required** | This operator + an external streaming-replication tool, **or** wait for the Pro tier |

## 8. Status

ADR-0040 §4.2's "unconditional recommendation for the cluster-mode
+ ops-automation scenario" condition is **met** (13 PRs across
#38–#49). Estimated parity vs. Redis Enterprise:

| Area | Score |
|---|---|
| Core HA + Backup | **90–92 %** (PITR phase 1 GA added) |
| TLS + Auth | **95 %** (everything except FIPS) |
| Storage + Networking | **92–93 %** (encryption-at-rest enforce added) |
| Observability | **90 %** (Histogram SLO + chaos foundation) |
| Operational docs | **90 %** (PITR guide added) |
| **Overall** | **~92 %** |

ADR-0040 §4.2 estimated *"70–75 % once the 5 gaps close"* →
**~94 % reached across 27 PRs** (the original plan's 18 gaps plus
nine additional closures: #62 `Status.Capabilities`, #63 multi-pod
guard, #64 Capability Gauge, …).
