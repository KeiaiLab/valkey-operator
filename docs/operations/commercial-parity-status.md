# Commercial Parity Status — valkey-operator

ADR-0040 (Helm chart vs Operator 채택 정책) 의 commercial parity 5 gap 해소 +
phase 2 추가 hardening 9건 반영 후의 상태. **production-grade OSS Redis/Valkey
operator** 로서의 기능 인벤토리.

비교 대상: Bitnami Redis chart (paywall 전환 후 OSS) / CloudPirates Redis chart /
Redis Enterprise / AWS ElastiCache / GCP Memorystore.

본 문서는 신규 사용자가 *"이 operator 로 production 운영 가능한가?"* 를 1분 안에
판단할 수 있도록 작성. 미달 항목은 *Pro tier / 후속 epic 후보* 로 명시.

## 8. Status

ADR-0040 §4.2 의 *Cluster mode + 운영 자동화 시나리오 unconditional 권장* 조건
**달성** (PR #38-#49 13개 완료). Redis Enterprise 대비 추정치:

| 영역 | 비교 |
|---|---|
| Core HA + Backup | **80~85%** (PITR 부재) |
| TLS + Auth | **95%** (FIPS 외 모두) |
| Storage + Networking | **90%** (encryption-at-rest enforce 외) |
| Observability | **90%** (Histogram SLO + chaos foundation) |
| 운영 자료 | **85%** (한국어 + 영어 mix, troubleshooting 풍부) |
| **Overall** | **~87%** |

ADR-0040 §4.2 의 *"5 gap 충족 시 70~75%"* 추정 → 실제 13 gap 해소로 **~87%** 도달.

## 9. 후속 epic 후보

본 문서의 §6 미달 항목 중 *commercial 가치 가장 큼*:

1. **PITR** (AOF replay-to-timestamp) — 단일 최대 차이
2. **VolumeSnapshot backup** — k8s native, 대용량 빠름
3. **Encryption at rest enforce** (audit 외 reject 옵션)
4. **HPA for Cluster mode shard scaling** (Track B Resharding 의존)

각각 별도 epic 으로 추적 권장 (#37 epic 은 closed).
